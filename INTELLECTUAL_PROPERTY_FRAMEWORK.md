## PART 5: IPFS INTEGRATION & BLOCKCHAIN STRATEGY

### 5.1 IPFS Architecture for Eagle Eye

**Purpose**: Create immutable audit trail for construction compliance findings

**Technical Implementation**:

```python
# Pseudo-code: IPFS-based audit trail

class IPFSAuditTrail:
    """
    Stores construction plan analysis results on IPFS for regulatory compliance.
    """
    
    def __init__(self, ipfs_client, pinata_key=None):
        """
        Initialize with local IPFS node + optional Pinata for redundancy.
        """
        self.ipfs = ipfs_client  # Local node
        self.pinata_key = pinata_key  # For 30-day retention guarantee
    
    def create_audit_record(self, analysis_result: dict) -> str:
        """
        Create immutable record of analysis.
        
        Returns: IPFS CID (Content Identifier)
        """
        # Create audit document
        audit_doc = {
            "timestamp": datetime.now(timezone.utc).isoformat(),
            "project_id": analysis_result["project_id"],
            "plan_hash": self.hash_file(analysis_result["plan_file"]),
            "findings": [
                {
                    "code": finding["code"],
                    "severity": finding["severity"],
                    "citation": finding["citation"],
                    "sheet_ref": finding["sheet_ref"],
                    "recommendation": finding["recommendation"],
                    "timestamp": finding["timestamp"]
                }
                for finding in analysis_result["findings"]
            ],
            "analyst": analysis_result.get("analyst", "system"),
            "status": "VERIFIED"  # Immutable proof of analysis completion
        }
        
        # Pin to local IPFS node
        cid = self.ipfs.add_json(audit_doc)
        
        # Pin to Pinata for 30-day redundancy
        if self.pinata_key:
            self.pinata_pin(cid, retention_days=30)
        
        return cid
    
    def verify_audit_record(self, cid: str) -> dict:
        """
        Retrieve and verify audit record from IPFS.
        """
        record = self.ipfs.get_json(cid)
        
        # Verify record hasn't been tampered with
        stored_hash = cid
        computed_hash = ipfs_hash(record)
        assert stored_hash == computed_hash, "Audit record tampered!"
        
        return record
    
    def generate_audit_report(self, project_id: str, date_range: tuple) -> dict:
        """
        Generate audit report for project (all analyses within date range).
        """
        audit_records = self.query_audit_trail(
            project_id=project_id,
            start_date=date_range[0],
            end_date=date_range[1]
        )
        
        return {
            "project_id": project_id,
            "total_analyses": len(audit_records),
            "findings_summary": aggregate_findings(audit_records),
            "audit_cids": [record["cid"] for record in audit_records],
            "report_cid": self.ipfs.add_json({
                "report_type": "audit_trail",
                "generated": datetime.now().isoformat(),
                "records": audit_records
            })
        }
```

### 5.2 Pinata Integration (Redundancy)

**Why Pinata?**
- Guarantees 30-90 day retention (IPFS nodes can garbage-collect)
- Provides HTTP gateway for easy verification
- Includes cryptographic proofs of pinning

**Implementation**:

```python
import requests

class PinataGateway:
    """
    Pinata service for IPFS redundancy and long-term storage.
    """
    
    def __init__(self, jwt_token: str):
        self.jwt = jwt_token
        self.base_url = "https://api.pinata.cloud"
        self.headers = {
            "Authorization": f"Bearer {jwt_token}",
            "Content-Type": "application/json"
        }
    
    def pin_to_ipfs(self, cid: str, retention_days: int = 30) -> dict:
        """
        Pin a CID to Pinata for guaranteed retention.
        """
        data = {
            "hashesToPin": [cid],
            "pinataMetadata": {
                "name": f"eagle-eye-audit-{cid[:8]}",
                "keyvalues": {
                    "retention_days": retention_days,
                    "system": "eagle-eye"
                }
            }
        }
        
        response = requests.post(
            f"{self.base_url}/pinning/pinByHash",
            headers=self.headers,
            json=data
        )
        return response.json()
    
    def verify_pin_status(self, cid: str) -> dict:
        """
        Check if CID is pinned and retrieval is guaranteed.
        """
        response = requests.get(
            f"{self.base_url}/data/pinList?hashContains={cid}",
            headers=self.headers
        )
        pins = response.json()["rows"]
        
        if pins:
            return {
                "pinned": True,
                "date_pinned": pins[0]["date_pinned"],
                "replicas": pins[0]["regions"],
                "status": "GUARANTEED"
            }
        return {"pinned": False, "status": "NOT_PINNED"}
```

### 5.3 Legal & Regulatory Implications of IPFS

**Advantages**:
- Immutable record for regulatory audits (EPA, local building dept)
- Tamper-proof chain-of-custody for insurance claims
- Decentralized storage = no single point of failure
- Cryptographic proof of analysis completion

**Challenges**:
- GDPR right-to-be-forgotten conflicts with immutability
  - **Solution**: PII redaction before IPFS pinning
  
- Data retention compliance varies by jurisdiction
  - **Solution**: Document retention policy (30-90 days standard)
  
- Blockchain/IPFS regulation still evolving
  - **Solution**: Consult legal before production use

**Recommended Approach**:
- Pin findings WITHOUT PII (project ID instead of client names)
- Maintain separate encrypted PII database
- Document IPFS usage in Terms of Service
- Enable user opt-out of IPFS pinning if desired
